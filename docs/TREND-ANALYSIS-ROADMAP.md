# 트렌드 분석 기능 로드맵

- **상태**: Phase 1~5 완료
- **목표**: 외부 트렌드 데이터를 수집·분석하여 콘텐츠 기획에 활용할 수 있는 "트렌드 분석" 탭 제공

## 아키텍처 요약

```
┌─────────────────────────────────────────────────────────────────┐
│                       트렌드 분석 탭                               │
├───────────────┬───────────────┬────────────────┬────────────────┤
│ 핫 토픽        │ 키워드 관심도   │ 정책/뉴스       │ 콘텐츠 추천     │
│ (RSS 기반)     │ (네이버 데이터랩)│ (RSS+네이버)    │ (교차 분석)     │
├───────────────┴───────────────┴────────────────┴────────────────┤
│ YouTube 인기 동영상 │ 트렌드↔성과 상관관계 │ 트렌드 변화 비교    │
└─────────────────────────────────────────────────────────────────┘

데이터 흐름:
  백그라운드 스케줄러 ──→ DB (trends 테이블) ──→ 프론트엔드 (캐시 읽기만)
  프론트엔드 ──→ 네이버 API (실시간 조회 OK, rate limit 여유)
  프론트엔드 ──✕──→ Google API (절대 직접 호출 안 함)
```

## 핵심 설계 원칙

### Google Trends 정책
- **프론트엔드에서 Google API 직접 호출 금지** — 사용자 클릭마다 요청 → rate limit 필연적
- Google Trends는 **백그라운드 스케줄러에서만 2시간 간격**으로 수집, 재시도 0회
- 성공하면 DB에 저장, 프론트엔드는 DB 캐시만 읽음
- 실패해도 RSS 핫 토픽이 기본이므로 UX에 영향 없음

### 데이터 소스 우선순위
| 우선순위 | 소스 | 용도 | 안정성 |
|---------|------|------|--------|
| 1 | RSS (Google News KR, 연합뉴스) | 핫 토픽, 뉴스 키워드 | 매우 높음 (키 불필요) |
| 2 | 네이버 데이터랩 | 키워드 관심도 차트 | 높음 (일 1,000건) |
| 3 | 네이버 뉴스 검색 | 카테고리별 뉴스 | 높음 (일 25,000건) |
| 4 | 정책브리핑 RSS | 정부 정책 뉴스 | 매우 높음 (키 불필요) |
| 5 | YouTube Data API v3 | 인기 동영상 | 높음 (일 10,000 유닛) |
| 6 | Google Trends | 급상승 검색어 (보너스) | 낮음 (비공식, rate limit) |

## 데이터 소스 상세

### 1. RSS 핫 토픽 (기본 트렌드 소스)
- **접근**: HTTP GET (인증 불필요, rate limit 없음)
- **소스**: Google News KR RSS + 연합뉴스 RSS
- **처리**: 헤드라인에서 키워드 빈도 추출 (언론사명/불용어 필터링)
- **갱신**: 앱 시작 시 + 자동 수집 시

### 2. 네이버 데이터랩
- **접근**: 네이버 개발자센터 Client ID/Secret (무료)
- **API**: `POST https://openapi.naver.com/v1/datalab/search`
- **제한**: 일 1,000건, 키워드 그룹 최대 5개 (그룹당 20개)
- **용도**: 키워드 관심도 비교 차트 (7일/30일/90일)

### 3. 네이버 뉴스 검색
- **접근**: 동일 Client ID 사용
- **API**: `GET https://openapi.naver.com/v1/search/news.json`
- **제한**: 일 25,000건
- **용도**: 카테고리별 최신 뉴스, 키워드 상세 패널

### 4. 정책브리핑 RSS
- **접근**: HTTP GET (키 불필요)
- **소스**: korea.kr (정책뉴스/보도자료/이슈인사이트)

### 5. YouTube Data API v3
- **접근**: Google Cloud Console API Key
- **API**: `GET /youtube/v3/videos?chart=mostPopular&regionCode=KR`
- **제한**: 일 10,000 유닛 (충분)

### 6. Google Trends (보너스)
- **접근**: `google-trends-api` npm (비공식 스크래핑)
- **제한**: 빈번한 rate limit, HTML 응답 반환
- **정책**: 백그라운드 2시간 간격, 재시도 0회, 실패 시 무시

## 기술 설계

### DB 스키마

```sql
trends         -- 트렌드 스냅샷 (source: google/hot-topics/policy/news/youtube)
trend_keywords -- 키워드 시계열 (keyword+source+date 유니크)
```

### 프로젝트 구조

```
collectors/trends/
  google-trends.js      # Google Trends (백그라운드 전용)
  naver-datalab.js      # 네이버 데이터랩 + 뉴스 검색
  rss-feed.js           # RSS 파서 + 핫 토픽 추출 + 키워드 빈도 분석
  youtube-trending.js   # YouTube 인기 동영상
src/js/
  12-trends.js          # 트렌드 분석 뷰
```

### API 키 관리 (설정 탭)

| API | 설정 필드 | 필수 여부 |
|-----|----------|----------|
| 네이버 데이터랩/뉴스 | `naverApi.clientId`, `naverApi.clientSecret` | 권장 |
| YouTube | `youtubeApiKey` | 선택 |
| Google Trends | 키 불필요 | — |
| RSS | 키 불필요 | — |
