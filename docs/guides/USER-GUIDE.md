# SNS 분석기 사용 가이드

SNS 분석기는 Threads, Instagram, TikTok 세 플랫폼의 SNS 성과를 통합 분석하는 Electron 데스크톱 앱이다. 포스트 성과 추적, 트렌드 분석, 콘텐츠 기획 지원까지 한 곳에서 관리할 수 있다.

---

## 시작하기

### 앱 설치 및 실행

1. 프로젝트 폴더에서 의존성 설치:
   ```bash
   npm install
   ```
2. 앱 실행:
   ```bash
   npm start
   ```
3. 앱이 시작되면 자동으로 1회 데이터 수집이 실행된다 (SNS + Google Trends + RSS).

### 초기 설정 (API 키 등록)

앱 실행 후 좌측 사이드바에서 **설정**(톱니바퀴 아이콘)을 클릭한다.

**SNS API 토큰 (Threads / Instagram)**

- SNS API 토큰은 자매 앱인 **card-news-maker**의 설정(`config.json`)에서 관리된다.
- card-news-maker에서 Threads/Instagram 토큰을 등록하면, SNS 분석기가 이를 읽기 전용으로 참조한다.
- 설정 화면의 "API 연결 상태" 테이블에서 각 플랫폼의 연결 여부를 확인할 수 있다.

**네이버 API (트렌드 분석용)**

1. [네이버 개발자센터](https://developers.naver.com/apps/#/register)에서 애플리케이션을 등록한다.
2. 설정 화면에서 Client ID와 Client Secret을 입력하고 "네이버 API 저장"을 클릭한다.
3. 키워드 관심도 비교, 카테고리별 뉴스 검색 등의 기능에 사용된다.

**YouTube API (트렌드 분석용)**

1. [Google Cloud Console](https://console.cloud.google.com/apis/credentials)에서 YouTube Data API v3 키를 발급한다.
2. 설정 화면에서 API Key를 입력하고 "YouTube API 저장"을 클릭한다.
3. YouTube 인기 동영상 조회 기능에 사용된다.

**TikTok**

- TikTok은 API 대신 uiautomator2 기반 에뮬레이터 스크래핑 방식으로 동작한다.
- 별도의 API 키 등록이 필요 없다.

## 기능별 가이드

### 1. 대시보드

사이드바 첫 번째 아이콘(집 모양)을 클릭하면 진입한다.

**요약 지표 카드**

화면 상단에 4개의 핵심 KPI가 표시된다:
- **총 팔로워**: 전체 플랫폼 합산 (플랫폼별 수치도 표시)
- **총 포스트**: 선택한 기간 내 포스트 수
- **총 조회수**: 선택한 기간 내 총 조회수
- **평균 참여율**: 포스트 평균 참여율

**기간 선택**

우측 상단의 기간 버튼으로 분석 기간을 변경할 수 있다:
- 7일 / 30일 / 90일

**팔로워 추이 차트**

- Threads, Instagram, TikTok 세 플랫폼의 팔로워 수 변화를 라인 차트로 보여준다.

**도달/노출 추이 차트**

- 각 플랫폼의 일별 도달 수를 Area 차트로 표시한다.

**포스트별 성과 차트**

- 최근 포스트의 좋아요/댓글/공유를 스택 바 차트로 보여준다.
- 상단 정렬 버튼으로 시간순, 좋아요, 댓글, 공유, 종합 순서로 정렬할 수 있다.
- 차트의 막대를 클릭하면 해당 포스트의 상세 페이지로 이동한다.

**핫 트렌드 위젯**

- 대시보드 하단에 현재 핫 토픽 키워드가 태그 형태로 표시된다.
- "트렌드 분석" 버튼을 클릭하면 트렌드 분석 화면으로 이동한다.

---

### 2. 콘텐츠 분석

사이드바 두 번째 아이콘(막대 차트)을 클릭한다.

**플랫폼 필터**

상단 탭에서 전체/Threads/Instagram/TikTok 중 하나를 선택하여 필터링할 수 있다.

**카테고리별 분석**

- **카테고리별 평균 참여도**: Grouped Bar 차트로 카테고리별 평균 좋아요/댓글/공유/저장을 비교한다.
- **카테고리별 포스트 수**: Doughnut 차트로 카테고리 구성 비율을 확인한다.
- **플랫폼 x 카테고리 비교**: "전체" 탭 선택 시 플랫폼별로 카테고리 성과를 비교하는 차트가 추가로 표시된다.
- **카테고리별 상세 테이블**: 포스트 수, 평균 조회/좋아요/댓글/공유 수치를 표로 확인한다.

기본 카테고리는 6개이다: 정부/정책, 금융/혜택, 트렌드, 라이프스타일, 기술/IT, 기타.

**해시태그 성과 분석**

카테고리 분석 아래에 해시태그 분석 섹션이 표시된다:
- **TOP 10 해시태그**: 평균 참여도 기준 상위 10개 해시태그 (가로 바 차트)
- **베스트 해시태그 조합**: 2회 이상 함께 사용된 해시태그 쌍의 평균 참여도
- **해시태그별 상세 성과 테이블**: 해시태그, 포스트 수, 평균 좋아요/조회/참여도 (헤더 클릭으로 정렬 가능)

---

### 3. 시간대/요일별 분석

사이드바 세 번째 아이콘(시계)을 클릭한다.

**히트맵**

- 가로축(0~23시) x 세로축(일~토) 격자로 표시된다.
- 셀의 색상 진하기가 해당 시간대의 평균 좋아요 수를 나타낸다.
- 셀에 마우스를 올리면 포스트 수와 평균 좋아요가 툴팁으로 표시된다.

**최적 발행 시간 TOP 5**

- 평균 좋아요가 가장 높은 요일+시간 조합 5개를 테이블로 보여준다.

**시간대별 포스트 분포**

- 24시간 기준 포스트 발행 빈도를 바 차트로 확인한다.

---

### 4. 교차 비교

사이드바 네 번째 아이콘(육각형)을 클릭한다.

동일한 콘텐츠를 여러 플랫폼에 게시했을 때, 플랫폼별 성과를 비교하는 화면이다.

**사전 준비**

- 동일 콘텐츠의 포스트에 같은 `content_group` 값을 지정해야 한다.
- 예: Threads와 Instagram에 같은 콘텐츠를 올렸다면, 두 포스트 모두 `20260314-oil-prices` 같은 동일한 그룹명을 설정한다.
- 2개 이상의 플랫폼에 동일 그룹이 있어야 비교가 가능하다.

**레이더 차트**

- 최근 5개 콘텐츠 그룹의 플랫폼별 종합 참여(좋아요+댓글+공유)를 레이더 차트로 비교한다.

**콘텐츠 그룹별 성과 테이블**

- 각 그룹에 대해 플랫폼별 좋아요/조회수를 나란히 표시한다.

---

### 5. 트렌드 분석

사이드바 다섯 번째 아이콘(상승 화살표)을 클릭한다.

외부 트렌드 데이터를 기반으로 콘텐츠 기획을 지원하는 종합 화면이다. 우측 상단 "새로고침" 버튼으로 전체 트렌드 데이터를 갱신할 수 있다.

**핫 토픽**

- Google News + 연합뉴스 RSS 기반으로 현재 핫 키워드를 보여준다.
- 카테고리 필터(테크, 경제, 정치, 라이프, 국제, 사회, 문화)로 관심 분야만 볼 수 있다.
- 키워드를 클릭하면 키워드 상세 패널이 열리며 관련 네이버 뉴스를 확인할 수 있다.

**키워드 관심도 비교**

- 네이버 데이터랩 API를 활용하여 최대 5개 키워드의 관심도를 라인 차트로 비교한다.
- 10분간 캐시되어 동일 요청 시 빠르게 응답한다.
- 네이버 API 키가 설정되어 있어야 사용 가능하다.

**키워드 인사이트**

- 각 키워드의 트렌드 방향을 표시한다: 급상승 / 상승 / 유지 / 하락 / 급하락.

**정책브리핑**

- korea.kr의 3개 RSS 피드에서 최신 정책 브리핑 기사를 수집하여 표시한다.

**뉴스 핫 키워드**

- RSS 헤드라인의 단어 빈도를 분석하여 워드 클라우드 형태로 보여준다.
- 언론사명은 자동으로 필터링된다.

**카테고리별 최신 뉴스**

- 네이버 뉴스 검색을 통해 7개 카테고리별 최신 뉴스를 제공한다.

**YouTube 인기 동영상**

- YouTube Data API v3를 사용하여 한국 인기 동영상 목록을 보여준다.
- YouTube API 키가 설정되어 있어야 사용 가능하다.

**콘텐츠 추천 + AI 아이디어**

- 현재 트렌드 키워드와 내 포스트 성과 데이터를 교차 분석하여 콘텐츠 아이디어를 추천한다.

**트렌드와 성과 상관관계**

- 트렌드 키워드를 반영한 포스트와 그렇지 않은 포스트의 평균 성과를 비교한다.

**콘텐츠 캘린더**

- 요일별 최적 발행 시간과 트렌드 키워드를 매칭하여 콘텐츠 발행 계획을 돕는다.

**트렌드 변화 비교**

- 우측 상단 "트렌드 변화" 버튼을 클릭하면 이전 스냅샷과 현재 스냅샷을 비교하여 새로 등장하거나 사라진 키워드를 확인할 수 있다.

### 6. 포스트 관리

사이드바 여섯 번째 아이콘(문서)을 클릭한다.

**포스트 목록**

- 전체 포스트를 테이블 형태로 보여준다.
- 상단 드롭다운으로 플랫폼별, 카테고리별 필터링이 가능하다.

**정렬**

- 테이블 헤더의 "좋아요", "조회", "발행일"을 클릭하면 해당 기준으로 정렬된다.
- 같은 헤더를 다시 클릭하면 오름차순/내림차순이 토글된다 (화살표 표시: ▼ 내림차순, ▲ 오름차순).

**포스트 추가**

- 우측 상단 "+ 포스트 추가" 버튼을 클릭하면 수동 입력 모달이 열린다.
- 입력 항목: 플랫폼, 캡션, 카테고리, 콘텐츠 그룹, 발행일시, 슬라이드 수, 해시태그
- 지표(조회수/좋아요/댓글/공유/저장/도달)도 선택적으로 입력할 수 있다.

**포스트 편집/삭제**

- 각 행의 "편집" 버튼으로 포스트 정보를 수정할 수 있다.
- 편집 시 새로운 지표 스냅샷을 추가할 수도 있다.
- "삭제" 버튼으로 포스트를 제거한다 (확인 대화상자 표시).

**카테고리 변경**

- 포스트 목록에서 카테고리 셀의 드롭다운을 직접 변경하면 즉시 저장된다.

**포스트 상세 분석**

포스트 행을 클릭하면 상세 페이지로 이동한다:
- **포스트 정보**: 캡션 전문, 해시태그, 게시일, 슬라이드 수, 콘텐츠 그룹
- **지표 카드 6종**: 조회수, 좋아요, 댓글, 공유, 저장, 참여율
- **지표 변화 추이 차트**: 좌측 Y축(좋아요/댓글/공유/저장) + 우측 Y축(조회수)의 Dual Axis 라인 차트
- **계정 컨텍스트**: 현재 팔로워 수, 도달률
- 좌측 상단 "뒤로" 버튼으로 이전 화면으로 돌아간다.

### 7. DM 관리

사이드바 일곱 번째 아이콘(말풍선)을 클릭한다. Instagram API 토큰이 설정되어 있어야 사용 가능하다.

**자동 DM 규칙**

댓글이 달리면 자동으로 DM을 보내는 규칙을 설정할 수 있다. 플로우는 다음과 같다:

1. **댓글 감지**: 게시물에 댓글이 달린다.
2. **팔로우 확인**: 댓글 작성자가 내 계정을 팔로우하는지 확인한다.
3. **미팔로우 시**: 팔로우 요청 메시지를 DM으로 발송한다.
4. **팔로우 확인 후**: 감사 메시지와 함께 정보를 DM으로 제공한다.

규칙 설정 방법:
- **트리거 조건**: "모든 댓글" 또는 "특정 키워드" 중 선택. 키워드는 쉼표로 구분하여 여러 개 입력 가능.
- **미팔로우 시 메시지**: 팔로우를 유도하는 메시지 입력.
- **팔로우 확인 후 메시지**: 실제로 전달할 정보/링크 등을 입력.
- "규칙 추가" 버튼으로 등록한다.

등록된 규칙은 목록에서 활성화/비활성화 또는 삭제할 수 있다.

**대화 관리**

- "대화 불러오기" 버튼을 클릭하면 Instagram DM 대화 목록을 불러온다.
- 대화를 선택하면 메시지 내역이 표시된다.
- 하단 입력란에 답장을 작성하고 Enter 또는 "전송" 버튼으로 발송한다.

**실시간 수신 로그**

- DM이나 댓글이 수신되면 화면 하단 로그에 실시간으로 표시된다.
- 최근 50건까지 유지되며, 타입(DM/댓글), 발신자, 내용, 시간이 표시된다.

> Webhook 서버는 port 3847에서 동작하며, ngrok 터널을 통해 Instagram과 연결된다.

### 8. 데이터 수집

사이드바 여덟 번째 아이콘(보관함)을 클릭한다.

**플랫폼별 수집**

- Threads, Instagram, TikTok 각각에 대해 "수집 시작" 버튼을 클릭하면 해당 플랫폼의 데이터를 수집한다.
- API 토큰이 미설정된 플랫폼은 버튼이 비활성화된다 (TikTok은 에뮬레이터 방식이므로 항상 활성).
- 각 카드에 마지막 수집 시간이 표시된다.

**자동 수집 설정**

설정 화면에서 자동 수집 스케줄러를 구성할 수 있다:
1. "자동 수집 활성화" 체크박스를 켠다.
2. 수집 간격(분)을 설정한다 (기본: 1분, 최소 1분 ~ 최대 1440분).
3. "스케줄러 저장"을 클릭한다.

자동 수집이 활성화되면 설정한 간격으로 SNS 데이터 + 트렌드 데이터가 반복 수집된다.

> 앱이 시작될 때는 자동 수집 설정과 관계없이 항상 1회 수집이 실행된다.

**수집 로그**

- 최근 20건의 수집 이력이 테이블로 표시된다.
- 각 행에 플랫폼, 상태(성공/실패/진행중), 메시지, 신규 포스트 수, 시작/완료 시간이 표시된다.

**급상승 키워드 알림**

- 수집 중 급상승 키워드가 감지되면 화면에 토스트 알림이 표시된다.

---

### 9. 주간 리포트

사이드바 하단의 문서 아이콘을 클릭한다.

최근 7일간의 성과를 요약한 리포트를 자동 생성한다.

**KPI 요약**

- 신규 포스트 수, 총 조회수, 총 좋아요, 팔로워 변화(증감)

**성과 TOP 3 포스트**

- 좋아요+조회수 기준 상위 3개 포스트 (클릭 시 상세 페이지 이동)

**카테고리별 성과 차트**

- 가로 바 차트로 카테고리별 조회수/좋아요를 비교한다.

**이번 주 트렌드 키워드**

- 핫 토픽 키워드 상위 6개를 태그로 표시한다.

**다음 주 추천**

- **최적 발행 요일**: 이번 주 데이터 기반 참여도가 높았던 요일
- **추천 키워드**: 트렌드 키워드 상위 3개
- **주력 카테고리**: 성과가 좋은 상위 2개 카테고리

**PDF 내보내기**

- 우측 상단 "PDF 내보내기" 버튼을 클릭하면 현재 리포트를 PDF 파일로 저장한다.

### 10. 설정

사이드바 아홉 번째 아이콘(톱니바퀴)을 클릭한다.

**API 연결 상태**

- Threads, Instagram, TikTok의 현재 연결 상태와 계정 ID를 확인한다.
- API 토큰은 card-news-maker 앱에서 관리되며, 여기서는 읽기 전용으로 표시된다.

**카테고리 관리**

- 기존 카테고리의 이름을 수정하거나 삭제할 수 있다.
- "새 카테고리 키" 입력 후 "추가" 버튼으로 새 카테고리를 등록한다.
- 변경 후 "카테고리 저장" 버튼을 클릭해야 반영된다.

**자동 수집 스케줄러**

- 자동 수집 활성화/비활성화
- 수집 간격(분) 설정

**네이버 API 설정**

- Client ID / Client Secret 입력

**YouTube API 설정**

- API Key 입력

**데이터 내보내기**

- "데이터 내보내기" 버튼으로 수집된 데이터를 외보할 수 있다.

## FAQ / 문제 해결

**Q: API 연결 상태가 "미설정"으로 표시된다.**

A: SNS API 토큰은 card-news-maker 앱의 설정(`config.json > snsAccounts`)에서 관리된다. card-news-maker에서 해당 플랫폼의 토큰을 먼저 등록해야 한다.

**Q: 트렌드 분석에서 "네이버 API 키를 설정하세요"라고 나온다.**

A: 설정 화면에서 네이버 개발자센터의 Client ID/Secret을 등록해야 한다. 키워드 관심도, 카테고리별 뉴스 등의 기능에 필요하다.

**Q: YouTube 인기 동영상이 표시되지 않는다.**

A: 설정 화면에서 YouTube Data API v3 키를 등록해야 한다. Google Cloud Console에서 API를 활성화하고 키를 발급받는다.

**Q: 교차 비교 화면에 "비교 가능한 그룹 없음"이라고 표시된다.**

A: 동일 콘텐츠를 2개 이상 플랫폼에 게시하고, 포스트 편집에서 같은 `content_group` 값을 지정해야 한다. 예: `20260322-spring-trends`.

**Q: 시간대 분석에 "데이터 없음"이 표시된다.**

A: 포스트의 발행 시간(`posted_at`)이 정상적으로 기록되어 있어야 한다. API 수집을 통해 자동으로 설정되지만, 수동 입력 포스트는 발행일시를 직접 입력해야 한다.

**Q: 자동 수집이 동작하지 않는다.**

A: 설정 화면에서 "자동 수집 활성화" 체크박스가 켜져 있는지 확인한다. 앱 시작 시 1회 수집은 설정과 무관하게 항상 실행되지만, 반복 수집은 활성화해야 한다.

**Q: DM 관리에서 "대화 없음 (검수 통과 후 사용 가능)"이라고 나온다.**

A: Instagram Messaging API는 Meta 앱 검수를 통과해야 실제 대화에 접근할 수 있다. 검수가 완료될 때까지 기다려야 한다.

**Q: 포스트 카테고리가 자동으로 지정되는 원리는?**

A: card-news-maker에서 제작한 프로젝트의 `category`와 `snsText`를 기반으로 `matchCategory()` 함수가 자동 매칭한다. 자동 매칭이 부정확한 경우 포스트 목록에서 드롭다운으로 직접 변경할 수 있다.

**Q: 데이터는 어디에 저장되는가?**

A: SQLite 데이터베이스 파일이 Electron의 userData 경로에 저장된다. macOS 기준 `~/Library/Application Support/card-news-sns-analyzer/` 경로이다.
